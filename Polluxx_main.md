# 🌟 OMNY-Twin (Polluxx) 模型架构摘要
*A4 Summary: EHR 基础模型工程架构*

---

## 🎯 定位与数据
- **类型**：EHR 纵向时序预测基础模型
- **目标**：建模长期健康轨迹 ➔ 预测未来临床事件
- **规模**：1.38B 总参数 | 322M 前向活跃 
  - *激活率 23.3% ➔ 4.3× 参数效率*
- **数据**：6,300 万条预分词医疗记录 (Max Len: 4096)

---

## 🏗️ 核心架构：[ Mamba × Attention × MoE ]
**拓扑**：混合型 Decoder-only | 共 16 层 | **[M, M, M, A] × 4** 循环

### 1️⃣ Mamba State Space (主干 75%)
- **机制**：$O(n)$ 线性时间复杂度
- **作用**：捕获跨月/年的纵向**长期时序依赖**

### 2️⃣ Attention 机制 (局部 25%)
- **机制**：低频调用，控制 $O(n^2)$ 算力爆炸
- **作用**：处理单次就诊 (Intra-visit) 内部的复杂事件交互

### 3️⃣ Sparse MoE (专家网络)
- **机制**：8 专家架构 | Top-2 动态路由 (25% 激活)
- **作用**：按病患特征自动激活特异性知识域 (心血管/神经等) | 零计算开销提升 4× 容量

---

## 🗺️ 极简数据流与拓扑结构

### 1. 数据管线 (Tokenization & Binning)
```text
[ 原始 EHR 记录 ] 
       │
       ▼
⚙️ 预处理与映射
 ├─ 静态特征 (年龄/性别) ────▶ [ Static Tokens (5个) ]
 ├─ 连续时间特征 (天数差) ───▶ [ ContinuousTimeEncoder (1280维向量) ]
 └─ 诊断/体征/检验/药物 ────▶ [ Clinical Tokens (严格预排序) ]
       │
       ▼
📦 序列装箱与特征融合
 [ 患者A: Static + <start> + [Enc]* + <end> ] + [ 患者B: ... ]
 (注: 连续时间特征直接 Add 至 Clinical Embedding，不独立占据 Sequence 槽位)
```

### 2. 模型处理管线
```text
[ 输入: Static + <start> + [Encounter]* + <end> ]
                            │
                            ▼
 [ Token Embeddings ]  ⊕  [ Continuous Time Encoder ] (1280维特征融合)
                            │
  ┌─────────────────────────┼─────────────────────────┐ 
  │ 🔄 循环 4 次            ▼                         │
  │     ╔═════════════════════════════════════╗       │
  │     ║ 🟢 Mamba Block 1 (线性特征抽取)     ║       │
  │     ║ 🟢 Mamba Block 2 (状态空间演进)     ║       │ 
  │     ║ 🟢 Mamba Block 3 (跨期纵向记忆)     ║       │
  │     ╚═══════════════════╤═════════════════╝       │
  │                         ▼                         │
  │     ╔═════════════════════════════════════╗       │
  │     ║ 🟠 Attention + Sparse MoE Block     ║       │
  │     ║  ├─ Multi-Head Attention (局部交互) ║       │
  │     ║  └─ MoE Router (8选2 专家路由)      ║       │
  │     ╚═════════════════════════════════════╝       │
  └─────────────────────────┼─────────────────────────┘
                            ▼
                  [ Final RMS Norm ]
                            │
             [ LM Head (与Embedding权重共享) ]
                            ▼
                   [ 预测 Next Token ]
```

## ⚡ 硬件与工程优化
- **张量对齐 (Tensor Core)**：词表极限填充至 `26,880` (64/128 倍数) ➔ GPU 吞吐量 ↑10-15%
- **显存压榨**：启用 Gradient Checkpointing ➔ 显存占用 ↓73% (~12GB/卡)
- **数值稳态**：RMSNorm (Pre-Norm) + bfloat16 ➔ 取消均值计算，防止梯度溢出
- **参数共享 (Tied Weights)**：LM Head 绑定 Embedding 权重 ➔ 原生统一输入输出空间，省去 34M 参数
