# 📘 附录 (Appendix)：工程落地与极限优化实录
*在 OMNY-Twin 模型从理论设计走向 4,000 万级真实医疗数据落地的过程中，我们直接遭遇并攻克了一系列大模型底层的“静默失败”与架构冲突。以下为核心排雷与优化打磨记录。*

---

### 🛡️ 优化项 1：彻底根治 Mamba 的状态毒化 (Padding Poisoning)

> **🚨 遇到的问题：特征稀释**
> 在处理长短不一的病历时，由于 Mamba 记忆力较长，它会将末尾堆砌的无数个 `<PAD>` 空白符作为有效状态摄入，导致病患早期真实的医疗特征被严重稀释（经测算，毒化率超过 **80%**）。

**🛠️ 我们的终极解法：**
1. **物理隔绝 (Sequence Packing)**：彻底废弃 Padding。采用 Sequence Packing，将海量短病历首尾相连，强行拼成长度为 `4096` 的标准序列，充分榨干显存。
2. **掩码重构 (Block Diagonal Mask)**：为了防止拼接后的不同病历产生“信息越权泄漏”（患者B读到患者A的诊断），我们在 Attention 层专门引入了**块对角因果掩码 (Block Diagonal Causal Mask)**。
3. **状态强制重置**：在每次序列中遇到新患者标签 `<patient_start>` 时，强制重置 Mamba 连续时间编码与底层的 Position ID，确保时序信息在患者间的**绝对隔离**。

---

### ⚖️ 优化项 2：根除 MoE 路由坍缩，防止高频词“被重复看诊”

> **🚨 遇到的问题：专家抢大户**
> 早期采用前沿的 Expert-Choice（专家选词）策略后，所有专家（专家网络）都倾向于“抢夺”高频、常规的体征指标。这导致一个高频 Token 被 6-8 个专家重复处理（梯度被异常放大 3 倍），而 **20%+ 的长尾罕见病 Token 无人问津**。

**🛠️ 我们的终极解法：**
1. **输出归一化 (Coverage Normalization)**：强行保留 140% 的 MoE 冗余总容量以兜底长尾召回。但在最终输出前除以 `coverage_count`，完美平抑高频特征引发的数值连环爆炸。
2. **全局限流 (Capacity Hard Limit)**：引入“带状态的顺序挑选逻辑”。一旦一个 Token 被选中达到 **3 次** ($K=3$)，我们立刻在算法底层将其对剩余专家的得分强行修改为 `-inf`，逼迫专家把计算名额留给低频罕见词。
3. **Z-Loss 保驾**：在路由分类器计算中引入 $Z_{loss}$ 正则化，抑制 Logits 极化，防止专家对某些词变得“盲目自信”。

### ⏱️ 优化项 3：化解同次就诊 (Intra-Encounter) 的时序坍缩冲突

> **🚨 遇到的问题：梯度迷失**
> 在同一次就诊记录中开出的多个医疗代码（例如同时开出的 5 个诊断），在时间戳上拥有**完全相同的 Time = 0**。这导致：Mamba 处理它们时缺乏明确的时序梯度指示，且与模型从左向右的自回归生成逻辑严重冲突。

**🛠️ 我们的终极解法：**
- **取消复杂数学，回归业务逻辑**：直接放弃了当时高昂但复杂的集合损失 (Set-based Loss) 研究。
- **数据级规范化排序 (Canonical Ordering)**：在极早期的预处理管线强行“立规矩”，要求所有**同次就诊的 Tokens 必须严格按医药逻辑定序**：
  👉 *诊断 (dx) ➔ 体征 (vs) ➔ 检验 (lb) ➔ 药物 (rx)*
  内部层级再按字典序绝对排列。
- **收益**：完美化解了 Teacher Forcing 机制对无序序列强加的不合理惩罚。通过这种强排序，模型就像自动学会了“医生写报告单表单填写的潜在规律”，流畅度剧增。

### 📉 优化项 4：识别数据管线与“虚假完美收敛”陷阱

> **🚨 遇到的问题：史诗级过拟合**
> 第一次全模型早期验证中，在仅仅 1000 步内，主 Loss 曲线就神勇地降至极其完美的 **0.0000** ($PPL=1.0$)。但同时观测到，MoE的专家网络当即发生了高达 **50% 级别的未覆盖坍缩**（大量专家直接停工）。

**🛠️ 我们的终极解法：**
1. **清醒诊断**：项目组没有被完美数字蒙蔽，敏锐查杀到这是“极小微型测试集导致的史诗级过拟合”。一旦 Loss 撞到底线0，梯度彻底消失，专家分化随即陷入停滞死亡。
2. **直接入海拦截干预**：果断否决部分 AI 优化工具关于“修改并复杂化模型架构”的误导性建议，**直接切入 4,000 万级真实海量病历数据流**暴力洗刷。
3. **参数强效护航**：为洪流制定了极其稳固的试运转堤坝（硬性锁定 Global Batch Size 为 `512`；学习率废弃自适应，死磕 `Warmup 预热 + Cosine 余弦衰减`）。最终在这波数据洪流的强力冲刷下，我们成功见证了底层专家的**自然苏醒与重新分化**，未覆盖率健康且稳步地降至 **10% 以内**。

### ⏳ 优化项 5：废弃离散 Gap Tokens，实现连续时间特征融合

> **🚨 遇到的历史局限：粗粒度的时间丢失与长度浪费**
> 过去我们采用将两次就诊的时间差强行“划分到 6 个离散的 Gap Tokens 分桶中”（如 `<gap_31-90>`）。这导致：
> - **精度受损**：模型无法区分“间隔了 31 天”与“间隔了 89 天”的复诊区别。
> - **序列浪费**：每一次时间间隔都硬性占用一个 4096 窗口内的宝贵 Token 位置，严重挤占了对真实临床事件记录的承载量。

**🛠️ 我们的终极解法：Continuous Time Positional Encoding**
1. **输入端降维打击**：彻底废弃离散 Gap Token 的 Embedding 表。现在数据管线会直接提取真实的连续浮点数天数差（如 `42.5` 天）`time_delta`。
2. **生成并直接相加融合**：引入专门的 `ContinuousTimeEncoder`（基于正弦/余弦基或 MLP），将这 `42.5` 天直接编码为与主干一致的 **1280 维**特征向量，随后**直接叠加相加 (Add) 到对应临床事件的 Token Embedding 上**。
3. **预测逻辑升维**：预测范式从“粗糙算日子”进化为了精准的介入式推理（即模型现在接受具体的未来时间点输入，来精准预测“如果 X 天后患者回来，会发生什么临床事件？”）。

**📈 核心收益：**
- **上下文扩容**：释放了海量无异议的 Gap Token 占位，使得单上下文可以容纳更加悠久的真实医疗序列跨度。
- **动态时序感知**：模型处理任何一个事件时，其底层 Embedding 都精准锚定了这距离上一次事件“究竟经过了几天几小时”，极大提升了长期纵向推理的细致度。
